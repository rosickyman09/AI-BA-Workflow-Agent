## Gateway Routing Rules (CRITICAL - DO NOT VIOLATE)

### Cookie & Auth Flow
ALL /api/* routes MUST proxy to Frontend (Next.js) first.
Next.js handles HttpOnly cookie <-> Bearer token conversion.

### Correct Nginx Routing (ALWAYS follow this):
```
  location /           -> proxy frontend:3000
  location /api/*      -> proxy frontend:3000  (Next.js handles cookie)
  location /api/auth/* -> proxy frontend:3000  (NOT auth_service)
  location /rag/*      -> proxy rag_service:5002
  location /auth/*     -> proxy auth_service:5001 (health only)
```

### FORBIDDEN Patterns (NEVER do this):
```
  location /api/auth/* -> proxy auth_service  X strips Set-Cookie
  location /api/*      -> proxy backend       X no token injection
```

### Verification Required After Any nginx.conf Change:
```
  Test 1: curl -c cookies.txt -X POST
          http://localhost/api/auth/login
          -> cookies.txt must have content
  Test 2: curl -b cookies.txt
          http://localhost/api/auth/me
          -> must return user info
  Test 3: Check response headers for Set-Cookie
          -> must NOT be empty
  If any test fails -> STOP and fix nginx.conf
```

## Async & Database Rules (CRITICAL)

### Promise.all() Restriction
NEVER use Promise.all() for unbounded database requests.
Maximum concurrent DB requests = 3 at any time.

### Required Pattern for Multiple DB Requests:
```typescript
  // CORRECT - batch of 3
  const batchSize = 3;
  for (let i = 0; i < items.length; i += batchSize) {
    const batch = items.slice(i, i + batchSize);
    await Promise.all(
      batch.map(item =>
        fetchData(item.id, { timeout: 5000 })
      )
    );
  }

  // WRONG - unbounded parallel (NEVER do this)
  await Promise.all(items.map(item => fetchData(item.id)));
```

### API Call Timeouts (MANDATORY)
```
  Frontend fetch calls:    5 second timeout
  Backend service calls:   10 second timeout
  DB queries:              30 second timeout
```

### PostgreSQL Connection Pool
- Max connections per service: 10
- Never exhaust pool with parallel requests
- Use batch processing for ALL bulk operations

### Loading State Rules
- Every async operation needs timeout handling
- Never leave user with infinite loading spinner
- Always show error message if request fails or times out

## Critical Development Rules
(Learned from Knowledge Base Fixes)

### Rule 1: Container Rebuild Required
After ANY code change:
  docker compose up -d --build [service]
  Never assume running container = latest code
  Health check (/health) does NOT verify
    all routes are updated
  Always rebuild after adding new routes/files

### Rule 2: JWT Claim Contract
All services MUST agree on JWT shape.
Documented contract:
  projects: [{project_id: string, role: string}]
  NOT: [string] (UUID only)
  NOT: {id: string} (wrong key)
When adding new service that reads JWT:
  Check auth_service JWT creation first
  Check existing services for consumption pattern
  Never assume claim format

### Rule 3: Next.js removeConsole Warning
next.config.js has removeConsole: true
  ALL console.* removed in production build
  Frontend debugging CANNOT use console.log
  Use error boundaries or UI error states
  Test with: docker logs ai_ba_frontend
  NOT browser console in production

### Rule 4: Checkbox + Label Event Bubbling
NEVER put <input type="checkbox">
directly inside <label>.
This fires click event TWICE.
Correct pattern:
  <div>
    <input
      type="checkbox"
      id="cb-{id}"
      onChange={handler}
    />
    <label htmlFor="cb-{id}">
      {text}
    </label>
  </div>

### Rule 5: React State Immutability
NEVER mutate state directly.
Sets and Arrays MUST return new reference.

WRONG:
  setSelectedDocs(prev => {
    prev.add(id)  ← mutates, no re-render
    return prev
  })

CORRECT:
  setSelectedDocs(prev =>
    new Set([...prev, id]))

  setItems(prev => [...prev, newItem])

### Rule 6: Stale Closures in Async
For async functions that need current state:
  Use useRef to mirror state:
    const selectedRef = useRef(selectedDocs)
    useEffect(() => {
      selectedRef.current = selectedDocs
    }, [selectedDocs])

  Use useCallback with deps array:
    const handleSearch = useCallback(async () => {
      const docs = selectedRef.current
      // use docs, not selectedDocs
    }, [])  ← empty deps, always current via ref

### Rule 7: Verify After Each Layer
When debugging multi-layer issues:
  1. Check DB data first
  2. Check backend API response
  3. Check Next.js proxy response
  4. Check frontend state
  Never skip layers - bugs stack on each other

### Container Rebuild Checklist
After any change, rebuild affected service:
  Backend change:
    docker compose up -d --build backend
  Frontend change:
    docker compose up -d --build frontend
  Both:
    docker compose up -d --build backend frontend

Verify ACTUAL code in container:
  docker exec ai_ba_backend
    cat /app/app/routers/[file].py | head -20
